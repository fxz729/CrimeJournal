# CrimeJournal 免费部署方案调研报告

> 调研日期：2026-03-20
> 项目类型：FastAPI后端 + React前端 + PostgreSQL数据库

---

## 一、GitHub Pages

### 核心限制
- **不支持后端API** - 仅支持静态内容（HTML/CSS/JS）
- **不支持数据库** - 无服务器端处理能力
- **不支持服务器端代码** - 不能运行Python/Node.js等后端语言

### 适用场景
- 纯静态网站
- 单页应用（SPA）前端托管
- 文档站点

### 结论
❌ **不适合CrimeJournal项目** - 无法托管FastAPI后端和数据库

## 二、免费数据库托管方案对比

### 1. Supabase（推荐用于全栈项目）

**免费额度：**
- 数据库存储：500 MB
- 文件存储：1 GB
- 认证用户：50,000 MAU
- Edge Functions：500K 调用/月
- 带宽：包含在内

**特色功能：**
- 内置认证系统（JWT、社交登录、MFA）
- 实时WebSocket订阅
- 文件存储 + CDN + 图片转换
- Row-Level Security (RLS)
- 自动生成的REST API

**付费计划：**
- Pro：$25/月（固定价格 + 超额费用）

**优势：**
- 完整的后端平台（数据库 + 认证 + 存储 + 实时）
- 一站式解决方案，减少集成工作
- 适合需要认证和实时功能的应用

**劣势：**
- 数据库分支功能较慢（基于迁移）
- 不支持自动休眠（Pro计划始终在线）

---

### 2. Neon（推荐用于纯数据库需求）

**免费额度：**
- 数据库存储：500 MB
- 计算：共享CPU，5分钟无活动后自动休眠
- 数据库分支：10个
- 带宽：包含在内

**特色功能：**
- **即时数据库分支**（毫秒级，写时复制技术）
- 自动休眠（无流量时零成本）
- 原生Serverless驱动（WebSocket连接）
- 与Vercel深度集成

**付费计划：**
- Launch：$19/月起（按使用量计费）
- 计算：$0.063/小时
- 存储：$0.16/GB

**优势：**
- 间歇性流量成本极低（自动休眠）
- 数据库分支速度快，适合CI/CD
- Vercel项目的最佳选择

**劣势：**
- 仅提供数据库，需自行集成认证/存储
- 冷启动延迟约0.4-0.75秒

---

## 五、推荐部署方案

### 方案A：完全免费方案（学生项目推荐）⭐

**架构：**
```
前端：Vercel (React)
后端：Render Free Tier (FastAPI)
数据库：Supabase Free (PostgreSQL)
```

**成本：** $0/月

**优势：**
- 完全免费
- Supabase提供数据库 + 认证 + 存储
- Vercel提供优秀的前端体验
- Render支持完整的FastAPI后端

**劣势：**
- Render后端会自动休眠（无流量时）
- 冷启动延迟（首次请求可能需要30-60秒）
- Render免费数据库30天后过期（需使用Supabase）

**适合场景：**
- 课程作业/演示项目
- 低流量应用
- 可接受冷启动延迟

---

### 方案B：低成本高性能方案

**架构：**
```
前端：Vercel (React)
```

**成本：** $5-10/月

**优势：**

**劣势：**
- 需要少量费用

**优势：**

**劣势：**
- 需要将FastAPI逻辑迁移到Edge Functions（Deno/TypeScript）

**适合场景：**

- 愿意使用Supabase生态系统

## 九、注意事项

### 1. 冷启动问题
- Render/Railway免费层会在无流量时休眠
- 首次请求可能需要30-60秒唤醒
- 解决方案：使用定时ping服务（如UptimeRobot）

### 2. 数据库连接
- 使用连接池（PgBouncer）避免连接数超限
- Neon和Supabase都提供内置连接池

### 3. 环境变量
- 在各平台配置环境变量（数据库URL、API密钥等）
- 不要将敏感信息提交到Git

### 4. CORS配置
- 后端需要配置CORS允许前端域名
- Vercel部署后会有固定域名
