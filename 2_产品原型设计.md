## 第二部分：产品原型 | Product Prototype

## 一、原型概述 | Prototype Overview

### 设计理念 | Design Philosophy

```
┌──────────────────────────────────────────────────────────────────┐
│  设计原则                                                        │
│  ─────────                                                       │
│  1. 简洁优先：首页突出核心功能（搜索 + 数据）                   │
│  2. 渐进式披露：高级功能通过二级菜单访问                       │
│  3. 响应式设计：移动端优先，适配多设备                         │
│  4. 无障碍支持：符合 WCAG 2.1 标准                            │
└──────────────────────────────────────────────────────────────────┘
```

## 六、原型设计总结 | Prototype Summary

### 关键设计决策 | Key Design Decisions

| 决策 | 选择 | 理由 |
|------|------|------|
| 首页布局 | 搜索优先 | 用户第一需求是搜索 |
| 搜索框 | 自然语言 | 降低学习成本 |
| 筛选器 | 左侧固定面板 | 便于多维度组合筛选 |
| 案例卡片 | 标题+摘要+标签+相关度 | 信息层次清晰 |
| AI 功能 | 案例详情页内 | 与案例上下文紧密结合 |
| 深色模式 | 全局支持 | 保护视力，夜间可用 |

### 设计原则落地 | Design Principles in Practice

```
✓ 简洁优先
  - 首页只有一个核心行动点：搜索
  - 搜索结果只显示最关键信息

✓ 渐进式披露
  - 高级筛选默认收起
  - AI 功能在详情页提供

  - Mobile: 单列布局，汉堡菜单
  - Tablet: 双栏布局，折叠侧边栏
  - Desktop: 固定侧边栏 + 主内容区

✓ 无障碍支持
  - 语义化 HTML
  - ARIA 标签
  - 键盘导航支持
  - 色彩对比度符合 WCAG AA
```
